# Исправление проблем на GitHub Pages

## Возможные проблемы и решения

### 1. Проблема: Белый экран
**Решение:** Убедитесь, что:
- `vite.config.js` содержит правильный `base: '/Technology-tracker/'`
- `src/App.jsx` использует `basename={import.meta.env.BASE_URL}`
- Файл `public/.nojekyll` существует
- После сборки создается `dist/404.html`

### 2. Проблема: API запросы не работают
**Решение:** 
- API запросы могут блокироваться CORS политикой
- Используйте режим тестовых данных (по умолчанию включен)
- Проверьте консоль браузера (F12) на наличие ошибок CORS

### 3. Проблема: localStorage не работает
**Решение:**
- localStorage работает на GitHub Pages
- Если не работает, проверьте настройки приватности браузера
- Код теперь обрабатывает ошибки localStorage gracefully

### 4. Проблема: Роутинг не работает
**Решение:**
- Убедитесь, что `dist/404.html` создается при сборке
- Проверьте, что файл `.nojekyll` существует в `public/`

## Проверка после деплоя

1. Откройте консоль браузера (F12)
2. Проверьте наличие ошибок
3. Проверьте, что все ресурсы загружаются правильно
4. Попробуйте перейти по разным маршрутам

## Если проблема осталась

1. Проверьте консоль браузера на GitHub Pages
2. Убедитесь, что сборка прошла успешно
3. Проверьте, что все файлы загружены в ветку `gh-pages`
4. Попробуйте очистить кеш браузера (Ctrl+Shift+R)
