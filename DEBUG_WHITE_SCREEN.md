# Отладка белого экрана на GitHub Pages

## Шаги для диагностики:

### 1. Проверьте консоль браузера (F12)
Откройте консоль разработчика и проверьте:
- Есть ли ошибки JavaScript?
- Загружаются ли файлы (JS, CSS)?
- Какие ошибки 404 (не найдены файлы)?

### 2. Проверьте Network вкладку
- Все ли файлы загружаются успешно?
- Правильные ли пути к файлам?
- Нет ли ошибок CORS?

### 3. Проверьте, что сборка прошла успешно
```bash
npm run build
```
Убедитесь, что:
- Создалась папка `dist/`
- В `dist/` есть `index.html` и `404.html`
- В `dist/` есть папка `assets/` с JS и CSS файлами

### 4. Проверьте base URL
В консоли браузера должно быть:
```
Base URL: /Technology-tracker/
Mode: production
```

### 5. Проверьте файл .nojekyll
Убедитесь, что файл `public/.nojekyll` существует (может быть пустым)

### 6. Проверьте GitHub Actions
- Запустился ли workflow?
- Прошла ли сборка успешно?
- Прошло ли развертывание?

## Частые проблемы:

### Проблема: Файлы не загружаются (404)
**Решение:** Проверьте, что `base` в `vite.config.js` правильный:
```js
base: '/Technology-tracker/'
```

### Проблема: Ошибки в консоли
**Решение:** Скопируйте ошибки и проверьте:
- Правильность путей к импортам
- Нет ли синтаксических ошибок
- Все ли зависимости установлены

### Проблема: Приложение не запускается
**Решение:** Проверьте `src/main.jsx` - там должно быть логирование в консоль

## После исправлений:

1. Соберите проект заново:
```bash
npm run build
```

2. Проверьте локально:
```bash
npm run preview
```

3. Закоммитьте и запушьте:
```bash
git add .
git commit -m "Исправление белого экрана"
git push
```

4. Дождитесь завершения GitHub Actions

5. Проверьте сайт через 1-2 минуты после деплоя
